How to use our grid

Framework-4's fantatic grid is based upon 12-columns that are contained within rows. You decide how many of the possible 12 columns you want a section to occupy and assign the corresponding column class. The column classes are given a left and right padding of 15px and the rows are given left and right negative padding of -15px.

The negative margins on the rows cancel out the left padding on the first column and the right padding on the last column in a row. If a row has more than 12 columns, the extra ones will float underneath and the spacing will still be uniform. Click on the buttons below for more on our grid structure and syntax.


	.gridContainer{
		padding: 0 15px;
		background-color: #fff;
	}
	.row{
		margin-right: -15px;
		margin-left: -15px;
	}

For our grid to work correctly, every page should have a div with a class of "container" inside the body tag. This div is used to limit the width of the page and to center the page in the browser window. Inside the "container" div, the page must have a "gridContainer." The header (with navigation) and the footer can be placed outside the "gridContainer" and will extend the full width of the "container" div.Inside the "gridContainer," you can use semantic sections like "main" and "aside".

Inside any semantic sections in the "gridContainer" you must have divs that have row and column classes.You can change the number of columns a div will occupy at different browser widths ("breakpoints"). These are set by media queries in the CSS and can be changed to suit your layout. It is good practice to make sure that the number of columns occupied on a row add up to 12 or multiples of 12 at every breakpoint. Your HTML code should should look like this, beginning with the body tag.


<body>
  <div class="container">
    <div class="gridContainer">
     <header>
     	<h4>Navigation </h4>
     </header>
     <main>
        <div class="row">
          <div class="col-s-12 col-m-6 col-l-3">Content</div>
          <div class="col-s-12 col-m-6 col-l-3">Content</div>
          <div class="col-s-12 col-m-6 col-l-3">Content</div>
          <div class="col-s-12 col-m-6 col-l-3">Content</div>
        </div><!--end row div-->
     </main>
    </div> <!--end gridContainer div-->
    <footer>
      <h4>Footer</h4>
    </footer>
  </div> <!--end container div-->
</body>

A column class begins with "col", identifies the width of the browser window (viewport) as small, medium, or large (s, m, or l) and specifies the number of columns that the div should occupy at that width. A div with a class of "col-s-6" will occupy 6 of the possible 12 columns (half the screen) at a small viewport. If you want the layout to change at various breakpoints, you will need to have a column style for each breakpoint. For exmaple, a div could take up 6 columns at small size, 4 columns at medium size, and 3 columns at large size. At very small size (mobile portrait), all divs default to 100%. If you want a section to be full width at all breakpoints, give it a single class of "col-s-12".

Our grid is based upon calculations made in CSS. If you would like to see the math, here are the formulas: 


.col-s-1{width: calc(100% / (12/1));}
